1、工程配置繁琐
2、遇到更新时，需重新替换
3、多个项目时需重复配置

在工程目录用终端命令就可以完成配置。

$ pod update --no-repo-update

#####创建pod私有库过程

1、 github上创建pod仓库SGNetworkService
https://github.com/wangpo/SGNetworkService.git

2、从github远端clone仓库SGNetworkService到本地

3、在文件目录SGNetworkService下创建Classes文件，将源代码拷入文件夹

4、在SGNetworkService目录下创建.podspec文件
bogon:SGNetworkService wangpo$ pod spec create SGNetworkService

5、然后用Xcode打开.podspec文件，进行编辑。编辑完成后，在终端可以输入
bogon:SGNetworkService wangpo$ pod lib lint --allow-warnings
此处可能会出现很多error 和 warning，根据提示重新打开.podspec文件进行修改，尤其注意需要引用依赖

6、为了对私有库进行维护更新，在SGNetworkService目录下创建一个demo工程，用pod指向本地库的形式执行pod install将Classes导入。测试工程是否正常运行。
pod 'SGNetworkService', :path => '../'

7、push本地仓库源代码到gihub远端，并打tag标签，与刚才编辑.podspec里面写的版本号一致。

8、github上创建私有仓库中心，管理自己pod库的podspec文件
https://github.com/wangpo/MyPrivateaPods.git

9、 本地创建私有库中心
pod repo add wangpoPrivatePodsCenter https://github.com/wangpo/MyPrivateaPods.git
创建成功后，可以进入~/.cocoapods/repos目录下可以看到wangpoPrivatePodsCenter文件

10、把SGNetworkService.podspec添加到私有库中心
bogon:SGNetworkService wangpo$ pod repo push wangpoPrivatePodsCenter SGNetworkService.podspec --allow-warnings

10、此时私有库制作完成，在Podfile文件中写入

运行pod install --no-repo-update(no Updating local specs repositories)
新的项目成功将私有库文件加载
12.私有库升级，重新编辑.podspec文件，修改相应的版本号，再次执行下面命令即可：
bogon:SGNetworkService wangpo$ pod repo push wangpoPrivatePodsCenter SGNetworkService.podspec --allow-warnings

13、本地私有库删除
到~/.cocoapods/repos目录下，通过以下命令行即可删除：
rm -rf wangpoPrivatePodsCenter
